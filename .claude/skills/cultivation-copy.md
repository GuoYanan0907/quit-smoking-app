# /cultivation-copy — 修仙文案顾问

自动扫描 Flutter 代码中的用户可见文案，将其优化为符合修仙主题风格的表达。

## 使用方式

```
/cultivation-copy <文件路径>
/cultivation-copy <代码片段>
```

## 执行流程

### 第一步：扫描识别

读取目标文件，识别所有用户可见的字符串，包括但不限于：

- `Text('...')` — 页面文字
- `AppBar(title: Text('...'))` — 标题栏
- `SnackBar(content: Text('...'))` — 提示信息
- `hintText: '...'` — 输入框提示
- `labelText: '...'` / `label: Text('...')` — 标签
- `ElevatedButton(child: Text('...'))` — 按钮文字
- `AlertDialog(title: Text('...'), content: Text('...'))` — 弹窗标题和内容
- `NavigationDestination(label: '...')` — 导航标签
- `showDialog` / `showModalBottomSheet` 中的文字
- `Icons.xxx` 旁边的描述文字
- 状态枚举的描述映射（如 `getStatusDescription` 返回值）

**不扫描的内容：**
- 变量名、函数名、类名
- 注释（`//` 和 `/** */`）
- 存储键名（如 `'quit_date'`）
- 日志输出（`debugPrint`、`print`）
- import 路径
- 技术标识符

### 第二步：风格评估

对照下方风格指南，判断每个文案是否需要优化：

**需要优化的情况：**
- 使用了通用/直白的表达（如"确定"、"取消"、"提交"、"保存"）
- 与修仙主题不协调的现代感措辞
- 缺少情感色彩的中性提示

**不需要优化的情况：**
- 已经符合修仙风格的文案
- 功能性极强、替换后会降低可理解性的文案（如日期"昨天"、"前天"）
- 数值和单位（如"20支"、"¥100"）
- 已经是修仙术语的文案

### 第三步：输出优化方案

对每个需要优化的文案，输出：

```
📍 位置：文件名:行号
   原文：「原始文案」
   建议：「优化文案」
   理由：简要说明优化思路
```

### 第四步：执行修改

等待用户确认后，使用 Edit 工具逐个替换文案。如果用户说"全部采用"则批量执行。

## 修仙风格指南

### 核心原则

1. **功能清晰优先**：修仙化包装不能牺牲用户理解
2. **适度克制**：不要每句话都堆砌修仙词汇，保持自然
3. **情感共鸣**：用修仙语境传达鼓励、支持、警示的情感
4. **一致性**：同类操作使用统一的修仙术语

### 特定场景文案模板

#### 首次使用
```
欢迎踏上修仙之路
二十一重境界，涤荡烟毒，重铸仙体
```

#### 境界突破
```
突破成功！
{境界名} — {称号}
修为大进，道心稳固
```

#### 心魔来袭
```
心魔来袭
{心魔名} 强度：★★★☆☆
道友，可要抵御此魔？
```

#### 戒烟成功
```
功德圆满！
二十一重境界，道友已尽数突破
烟毒已除，仙体已成
```

#### 空状态
```
灵台空明，暂无记录
道友，请开始今日修行
```

#### 确认对话框
```
标题：{操作名称}
内容：{操作描述}，道友确定要继续吗？
按钮：[罢了] [遵命]
```

### 文案长度建议

- 按钮文字：2-4 个字
- 提示信息：不超过 20 个字
- 弹窗标题：不超过 10 个字
- 弹窗内容：不超过 50 个字
- 空状态描述：不超过 15 个字

### 保持一致性的关键

以下文案在整个 APP 中应保持统一：
- 取消按钮统一用"罢了"或"取消"（二选一，全 APP 一致）
- 确认按钮统一用"善"或"确认"（二选一，全 APP 一致）
- 对用户的称呼统一用"道友"
- 成功提示统一用"功德圆满"或"施法成功"（二选一，全 APP 一致）
- 错误提示统一用"走火入魔"或"修行受阻"（二选一，全 APP 一致）
